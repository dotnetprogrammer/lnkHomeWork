about IE

인터넷 익스플로러(이하 "IE")는 버전 별 지원하는 css의 속성에 차이가 있음.
parallax 스크롤을 구성하는 대부분의 css3 요소가 IE 10 이상부터 작동함.
즉, 하위 버전 IE에서의 티져사이트 작동 여부를 보장할 수 없거나 제한적으로 됨.

parallax 스크롤의 기법 중 "뭔가 회전하고 주기적으로 움직이는"속성은 IE 9 이하에서는 잘 안되는 것이 대부분임.

모션별 IE 구현 가능성 판별 방법

  • 객체가 이동하는 경우
    객체가 이동하는 모션은 transform 속성을 사용할 수 있으나 전통적으로 position 속성을 통해 사용할 수도 있음.
    화면이나 객체가 2차원적으로 움직일 경우(상하좌우) 50%의 확률로 나눌 수 있으며 정확한 것은 구현방법을 들여봐야 함.
    사용하는 plug-in이 전통적인 방법이 아닌 transform 속성만 지원한다면 대체할 방법을 찾는 것이 어려움.
    scroll path의 경우 전체적으로 화면이 2차원(상,하,좌,우)으로 움직이는 것은 가능하지만 4에서 5로 넘어가는 효과와 5에서 6으로 넘어가는 효과는 하위버전 IE에서 구현되지 않음.
    IE 9이상 부터 가능(IE 9에서는 -ms- vender prefix 사용됨.) 브라우저 지원현황
  • 지속적으로 움직이는 경우
    css3가 보편화된 상황에서는 animation이라는 속성을 사용하지만 기존의 전통적인 방법으로도 구현할 수 있음.
    사용하는 plug-in이 지원하지 않을 경우 대체가 불가능하다고 봐야함.
    자사 사이트처럼 css의 박스모델을 통한 구현은 가능하지만 다양한 기능을 제공하는 plug-in에서는 지원하지 않음.
    예제 사이트처럼 작동하는 예제의 경우는 하위버전 IE에서 구현이 어렵다고 생각해야 함.
    IE 10이상 부터 가능 브라우저 지원현황
  • 객체가 회전하는 경우
    회전하는 객체는 다른 css 속성으로 대체 되는 경우가 없음.
    scroll path의 4~5~6 구간
    예제 사이트의 우측 박스 회전같은 모션.
    IE 10이상 부터 부분적(partial)으로 가능 브라우저 지원현황
상황 구현방법 설명 예제 사이트 브라우저 지원 현황 비고
객체가 이동하는 경우 css 3 : transform(2d)
css 2.1 : left, top, right, bottom 값들을 조절하여 구현도 가능함.		 화면 상에서 객체가 2차원으로 움직이는 상황을 구현할 때 사용. scroll path의 경우 전체적으로 화면이 2차원(상,하,좌,우)으로 움직이는 것은 가능하지만 4에서 5로 넘어가는 효과와 5에서 6으로 넘어가는 효과는 하위버전 IE에서 구현되지 않음. IE 9이상 부터 가능(IE 9에서는 -ms- vender prefix 사용됨.) 브라우저 지원현황 단, 플러그인 css 3의 방법으로 구현할 경우 css 2.1로 대체할 방법이 없음.
객체가 회전하는 경우 css 3 : transform(3d)
css 2.1 : 없음.		 화면 상에서 객체가 3차원으로 움직이는 상황을 구현할 때 사용. scroll path의 4~5~6 구간
예제 사이트의 우측 박스 회전같은 모션.		 브라우저 지원현황 현재 webkit 계열 브라우저만 됨.
객체가 지속적으로 움직이는 경우 css 3 : animation
css 2.1 : left, top, right, bottom, margin 값들을 조절하여 구현도 가능함.		 화면 상에 객체가 반복적, 지속적으로 움직이는 상황을 구현할 때 사용. 단, 사이즈가 변하는 모션은 제한적 상황이 많음. 자사 사이트처럼 css의 박스모델을 통한 구현은 가능 브라우저 지원 현황 단, 플러그인 css 3의 방법으로 구현할 경우 css 2.1로 대체할 방법이 없음.
